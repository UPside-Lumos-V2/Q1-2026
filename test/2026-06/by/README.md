## Root cause analysis

- **Title**: Public BYToken auto-burn can burn AMM pool balances and sync manipulated reserves
- **Severity**: `critical`
- **Confidence**: `high`
- **Violated invariant**: A public token-maintenance function must not let arbitrary callers burn AMM pool-owned tokens and sync AMM reserves outside authorized liquidity/accounting operations.

### Final root cause

BYToken exposes triggerAutoBurn(), which unconditionally calls _autoBurn(true). When the calculated burn exceeds the token contract's own BY balance, _autoBurn(true) burns the remaining amount from the configured AMM pool and immediately calls PancakePair.sync(), letting any caller reset reserves after reducing the pool's BY balance. The attacker triggered this branch inside the flash-loan callback, then used router swaps under the manipulated reserve baseline to drain WBNB/BNB.

### Recommended fixes

- Change BYToken triggerAutoBurn() so the public path cannot pass allowPoolBurn=true; reserve pool burns and subsequent pair.sync() for authorized, bounded maintenance only.
- Add an explicit invariant check before any pool burn that preserves AMM/LP value or forbids burning pool-owned tokens entirely outside controlled liquidity operations.
