## Root cause analysis

- **Title**: Accepted Wormhole transfer VM authorized an unbacked wrapped ALPH mint
- **Severity**: `critical`
- **Confidence**: `medium`
- **Violated invariant**: A wrapped-token mint must only occur for a unique, validly guardian-signed cross-chain transfer message from the registered emitter whose amount and recipient correspond to a legitimate source-chain burn or lock entitlement.

### Final root cause

Bridge._completeTransfer accepted a Wormhole VM as valid and, for a foreign token, minted wrapped ALPH to the attacker through TokenImplementation.mint. The token mint itself was owner-authorized because the bridge is the wrapped token owner, so the failed economic decision is the upstream VM/bridge entitlement check rather than an ERC20 transfer or approval path. The exact vulnerable Wormhole verifier branch is not source-backed in the supplied artifacts, so the RCA is partial.

### Recommended fixes

- Add cross-chain transfer accounting tests that compare every wrapped-token mint against a unique accepted source-chain transfer entitlement.
